Pattern · Click-to-copy chips
Effort: small Impact: reaches every lesson Stripe Docs · Tailwind Docs

Every inline <code> chip becomes tappable. Tap → instantly copies to clipboard → tiny ✓ toast. Zero friction reward for engaging with examples.

Try tapping any chip below:
Use .join() instead of += in a loop. Convert with int("5") or "5" + str(3). The pattern f"{x=}" prints both name and value.

Dopamine: ~80ms haptic-feeling toast. Pure-learning anchored — copying example code is THE primary mode of engagement in a coding lesson.
Implementation

Pattern · Inline predict prompts
Effort: medium Impact: every coding lesson Roediger & Karpicke 2006 · g≈0.50

Mid-lesson, when a concept is about to be revealed, show a 3-option multiple-choice prompt FIRST. Answer → instant ✓/✗ + +5 XP + revelation. This is the testing effect — the single highest-leverage retention mechanic in cog-science.

What does 0.1 + 0.2 == 0.3 evaluate to in Python?
+5 XP Floats are binary. 0.1 + 0.2 == 0.30000000000000004 in IEEE 754. For money, use decimal.Decimal.
Dopamine: active prediction → instant confirmation. Wrong answers DON'T penalize XP — exposing the misconception is its own win. ~3 of these per lesson, max.
Pattern · Tap-to-reveal answers
Effort: small Impact: glossary-style lessons Active recall · Anki UX

Convert flat factual content into curiosity gaps. The question is visible; the answer is hidden behind a button. Spaced-rep at the lesson level.

What's the time complexity of list.pop(0) in Python?
O(n). Python lists are backed by contiguous arrays — popping the first element forces every other element to shift left. Use collections.deque for O(1) popleft.
When does a Python dict lookup degrade from O(1) toward O(n)?
When hash collisions pile up on the same bucket — usually only under adversarial inputs (deliberate hash-flooding attacks) or a buggy __hash__ implementation. Python's open-addressing handles benign collisions in near-O(1) amortized.
Dopamine: classic "I knew that!" moment when the answer matches your guess. New block type { type: 'reveal', q, a }. Authoring lift: minimal — drop into any existing lesson.

Effort: medium Impact: any "common bug" pattern Stack Overflow culture · code review intuition

Show buggy code with the bug highlighted. Tap the bug → it transforms into the correct version with a satisfying micro-animation. Each lesson can have 2-3 of these embedded.

Pattern · Runnable code blocks inline
Effort: medium Impact: every static code block Codecademy · ACM SIGCSE 2023

Every code block in a lesson gets a small "▶ Run" pill. Tap → expands to an inline mini-REPL using existing Pyodide/sandbox runtime. No leaving the lesson, no separate IDE setup. Apply the concept where you read it.

A list comprehension with a filter:

nums = [1, 2, 3, 4, 5, 6]
evens = [n for n in nums if n % 2 == 0]
print(evens)
▼ OUTPUT
[2, 4, 6]
Why it wins: Removes the "set up Python" cliff between reading and doing. Reuses already-shipped PracticeBlock + Pyodide. Every static code block becomes a try-it moment. Codecademy's structural lift over standard MOOC completion (~5%).
